## Quick checklist
- Provide three example usage profiles.
- Define the unit in plain language.
- Offer a simple estimation worksheet.
- Show the plan that best fits each profile.
- Update examples quarterly.

## Step-by-step
1. Define the primary unit and a simple formula.
2. Build low, medium, and high usage examples.
3. Map each example to a plan or tier.
4. Include a short FAQ for estimation questions.
5. Add a support contact for edge cases.

## What to watch
- Examples that do not match real customers.
- Units that are too technical to estimate.
- Estimation tools that are not mobile-friendly.

## Common mistakes
- Over-optimizing for the median user only.
- Hiding overage pricing in a separate page.
- Changing units without updating examples.
